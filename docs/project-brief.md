# 먹뛰기 (Ink Jump) — 프로젝트 브리핑

> 이 문서는 외부 AI 도구(ChatGPT 등)에게 프로젝트 컨텍스트를 전달하기 위한 종합 브리핑
> 자료입니다. 게임 기획부터 현재 구현 상태까지 이 문서 하나로 파악할 수 있도록 작성했습니다.

## 1. 한 줄 소개

> "선 하나가 발판이 되고, 발판 하나가 그림이 된다"

NHN NAN 2026 · Game × AI Hackathon 제출작. Unity 기반 모바일(Android) 세로 화면
드로잉 클라이밍 게임. 팀명 Team-Swooord, 2인 팀.

## 2. 게임 개요

| 항목 | 내용 |
|---|---|
| 장르 | 캐주얼 아케이드 클라이밍 (드로잉 플랫포머, 점프킹 계열) |
| 플랫폼 | 모바일 Android APK, 세로 화면 |
| 엔진 | Unity 6 (6000.3.10f1), URP, 2D |
| 타겟 | 전 연령, 캐주얼 퍼즐·아케이드 유저 |

| 코어 루프 | 관찰 → 발판 스케치 → 자동 점프 → 상승/추락 → 재도전 |

## 3. 핵심 메커닉

- 캐릭터(먹방울이)는 **일정 주기마다 자동으로 점프**한다. 플레이어는 점프 자체를
  조작할 수 없다.
- 플레이어는 화면 하단(한지 여백)에 **손가락으로 선을 그어 발판을 생성**한다.
- 발판의 **위치·각도·길이**가 다음 점프의 궤적(방향, 높이)을 결정한다.
- 그린 발판에는 가파른 대각선에서도 붙을 수 있지만 화면 양옆 경계는 반동 벽으로 유지된다.
- 발판을 잘못 그리거나 타이밍을 놓치면 캐릭터가 굴러 떨어진다.
- 전략의 핵심은 "내 점프 컨트롤"이 아니라 **"그림으로 만드는 길"**.
- 플레이어가 대충 그린 스케치는 AI가 실시간으로 수묵화 붓질 질감의 발판으로 완성한다
  (해커톤 데모의 핵심 차별점). API 실패/미설정 시에도 절차적 먹 텍스처 폴백으로
  동일하게 플레이 가능해야 한다 (심사자가 API 키 없이 실행 가능해야 하는 제출 요건).

## 4. 캐릭터 — 먹방울이

센과 치히로의 숯검댕이(스스와타리)를 동양화 먹빛으로 재해석한 캐릭터.

**비주얼 스펙 (확정)**
- 몸통: 먹 번짐 텍스처의 불규칙한 스파이크형 검은 원 (붓 터치로 그린 잉크 블롯 실루엣)
- 눈: 크고 둥근 흰색 타원 + 검은 원형 동공. **눈썹·입·안광(하이라이트) 없음** —
  표정은 눈 모양과 다리 포즈로만 표현
- 다리: 팔 없이 다리만 존재. 기본(대기) 포즈는 무릎이 몸통 옆으로 뾰족하게 튀어나온
  산(山) 모양 실루엣 — 몸통 아래에서 위로 짧게 붙었다가 몸통 바깥쪽 높은 지점(무릎)
  까지 솟은 뒤, 다시 아래 발까지 길게 떨어지는 꺾인 선. 발끝에는 작은 훅 모양 표시
- 그림자 없음 (땅 그림자는 게임 내 조명/이펙트로 별도 처리)
- 원본 파일: `Assets/Art/Character/Player/character_muk_bangul_v3.png` (1024×1024, 투명 배경)

**애니메이션 상태 (구현 필요)**
- Idle / 점프 대기
- 점프 중 (웅크림 → 도약 → 정점 → 하강)
- 착지
- 낙하/실패

## 6. 배경 아트

- 세로 산수화 (1080×1920), 파일: `Assets/Art/Background/background_ink_landscape.png`
- 하단 약 620px(`y: 1300~1920`)은 의도적으로 비운 "한지 여백" — 실제 발판을 그리는

  설산으로 배경 화풍 자동 전환

## 7. 현재 구현 상태 (2026-07-20 기준)

Unity 프로토타입의 **Week 1 코어 루프 + 첫 라운드 버그 수정까지 완료**되어 main
브랜치에 머지됨. 실제 에디터 플레이 테스트를 거쳐 여러 버그를 수정한 상태.

**완료된 기능**
- 자동 점프: 첫 점프는 접지 중 충전하고, 이후에는 점프 정점부터 다음 충전을 시작한다.
  발판 표면 노멀이 점프 방향을, 발판 길이가 점프력을 결정 (`AutoJump.cs`)
- 드로잉 발판: 터치/마우스 스트로크 캡처 → Chaikin 스무딩 → 리샘플링 → EdgeCollider2D
  발판 생성 (`StrokeCapture.cs`, `BezierSmoother.cs`, `PlatformCollider.cs`)
- 발판은 양방향 실선 충돌, 대각선 접착 지원, 수명 6.5초 페이드, 동시
  최대 4개, 먹 잔량(획 하나 최대 길이 6유닛), 캐릭터 반경 내 획 무효화(물리 밀어내기
  악용 방지)
- 발판 비주얼은 Main UI의 `LineSprite` 텍스처를 LineRenderer 붓결로 사용하고,
  해당 오브젝트가 없으면 절차적 갈필 텍스처로 폴백한다.
- 장애물: 좌우 이동 먹가시와 예고 후 수직 낙하하는 낙묵석. 낙묵석은 드로잉 발판을 파괴한다.
- 아이템: 먹물방울(50m 점프), 황금 붓(8초 무한 먹), 먹 방어막(피해 또는 추락 1회 방지).
  Codex로 생성한 자체 VFX·SFX를 사용한다.
- 로비: 저장된 최고 고도와 로컬 랭킹 팝업을 표시한다.
- AI 수묵 변환 골격: `SketchToInkService.cs` — 폴백(런타임 생성 갈필 텍스처)이
  즉시 적용되고, 원격 img2img API 훅은 비동기로 추가 예정 (Week 2, 아직 미구현).
  API 키 없이도 완전히 플레이 가능한 구조
- 게임 루프: 위로만 따라가는 카메라, 고도 점수 + 최고 기록(PlayerPrefs), 추락 →
  터치 재도전
- 입력: Device Simulator/터치/마우스/펜을 모두 지원하는 `PointerInput` 헬퍼
- 씬은 코드로 자동 구성 (`MukJumpSceneBuilder.cs`, 메뉴 `MukJump > Build Main Scene`)
  — 협업 시 씬 파일 머지 충돌 방지

**진행 중**
- 캐릭터 점프 애니메이션 (8프레임으로 제작 중)

**미구현 (로드맵)**
- 원격 AI 수묵 변환 API 실제 연동 (현재 폴백만 동작)
- 고도별 배경 전환, 적응형 국악 사운드, 개인화 낙관 등 확장 기능
- Android 빌드 서명·실기기 테스트
- 최종 제출 문서 3종(게임 소개서/AI 활용 기술 문서/팀원 롤 기술서)
